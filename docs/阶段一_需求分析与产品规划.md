# 交付物一：用户画像与场景调研（P0-01）

## 1.1 市场背景

全球移动游戏市场在 2025 年达到约 **504 亿美元**规模，预计到 2035 年将增长至 **540.5 亿美元**（CAGR 2.3%）。移动端占全球游戏总收入的 60% 以上。智能手机用户的核心活动分布中，**流媒体娱乐占 40%，游戏占 25%**，两者合计占据用户手机使用时长的 65%。

> 数据来源：Global Growth Insights, Tekrevol Mobile App Statistics 2026

**关键趋势**：
- 生成式 AI 正在改变移动游戏的开发与消费模式
- 多人社交/聚会类游戏需求持续增长
- 家庭共享娱乐场景（大屏+多设备）成为新兴赛道
- 跨平台一致性体验成为用户基本期望

## 1.2 目标用户画像

### 画像 A：家庭管理者（家长）

| 属性 | 描述 |
|------|------|
| **年龄** | 30-45 岁 |
| **角色** | 家庭活动组织者，通常是母亲或父亲 |
| **设备** | 主力手机（iOS/Android）+ 平板 + 智能电视 |
| **痛点** | ① 假期不知如何安排家庭活动；② 担心孩子沉迷手机/不当内容；③ 缺少适合全家参与的互动方式；④ 多个娱乐 App 分散管理困难 |
| **需求** | 一站式家庭娱乐平台、儿童内容安全管控、活动规划工具、家庭成员互动 |
| **使用场景** | 周末/假期组织家庭活动、孩子课后时间管理、家庭聚会娱乐 |
| **技术熟练度** | 中等，偏好简洁直观的操作界面 |

### 画像 B：青少年用户（孩子）

| 属性 | 描述 |
|------|------|
| **年龄** | 8-16 岁 |
| **角色** | 家庭娱乐的主要参与者 |
| **设备** | 自己的手机/平板（家长管控下） |
| **痛点** | ① 觉得家庭活动无聊；② 想和父母一起玩但缺少合适工具；③ 独自玩手机被父母批评 |
| **需求** | 有趣的互动游戏、与父母/兄弟姐妹的竞赛、获得成就感和认可、自主选择活动 |
| **使用场景** | 假期空闲时间、家庭聚会、与兄弟姐妹互动 |
| **技术熟练度** | 高，快速上手新 App |

### 画像 C：祖辈用户

| 属性 | 描述 |
|------|------|
| **年龄** | 55-70 岁 |
| **角色** | 家庭活动的被动参与者，希望融入家庭互动 |
| **设备** | 智能手机（大屏机型为主） |
| **痛点** | ① 不熟悉复杂操作；② 难以融入年轻人的数字娱乐；③ 视力/操作不便 |
| **需求** | 简单易用的界面、大字体/大按钮、语音操作支持、适合老年人的休闲游戏 |
| **使用场景** | 家庭聚会、节假日团聚 |
| **技术熟练度** | 低，需要极简设计 |

## 2.2 详细竞品分析

### 2.2.1 Jackbox Party Pack

| 维度 | 分析 |
|------|------|
| **产品定位** | 多人派对游戏合集，一人购买全家参与 |
| **核心功能** | 问答（Quiplash）、画猜（Drawful）、文字欺骗（Fibbage）等 5 款/包 |
| **优势** | ① 游戏品质极高；② 手机作为控制器，无需额外下载；③ 内容幽默风趣；④ 适合 2-8 人 |
| **劣势** | ① 需要主机/PC 作为主屏幕；② 付费制（每包约 $30）；③ 缺少家庭管理功能；④ 无中文支持 |
| **用户评价** | 聚会游戏品类标杆，但硬件门槛限制了家庭场景的便捷性 |
| **可借鉴** | 房间码加入机制、游戏类型多样性、幽默内容设计 |

### 2.2.2 Netflix Games

| 维度 | 分析 |
|------|------|
| **产品定位** | 流媒体订阅制下的附加游戏服务 |
| **核心功能** | 独占游戏（Lego Party!、Tetris Time Warp 等）、与影视 IP 联动 |
| **优势** | ① 订阅制零额外成本；② 强 IP 背书；③ 与影音内容无缝整合 |
| **劣势** | ① 游戏数量和质量参差不齐；② 缺少社交互动功能；③ 无家庭空间概念 |
| **用户评价** | 作为 Netflix 附加功能尚可，但无法替代专业聚会游戏 |
| **可借鉴** | 订阅制商业模式、影音+游戏整合思路 |

### 2.2.3 Heads Up!

| 维度 | 分析 |
|------|------|
| **产品定位** | 轻量级猜词聚会游戏 |
| **核心功能** | 举手机到额头猜词、录屏分享、自定义词库 |
| **优势** | ① 上手极快；② 录屏分享社交传播力强；③ 支持多语言 |
| **劣势** | ① 单一游戏类型；② 缺少长期留存机制；③ 无多人远程联机 |
| **用户评价** | 聚会破冰神器，但耐玩性有限 |
| **可借鉴** | 极简上手体验、录屏分享机制、自定义内容 |

### 2.2.4 国内家庭娱乐类 App

| 维度 | 分析 |
|------|------|
| **代表产品** | 全民K歌、家庭K歌、各种聚会小游戏 |
| **优势** | ① 深度本地化；② 社交链整合（微信）；③ 免费模式 |
| **劣势** | ① 功能单一（仅 K 歌或仅游戏）；② 缺少家庭场景整体方案；③ 儿童保护不足 |
| **可借鉴** | 微信生态整合、本地化内容、免费+增值模式 |

## 2.4 竞争策略总结

**差异化定位**：市场上缺少一款将"多人互动游戏 + 影音娱乐 + 家庭活动规划"三者整合的一站式家庭娱乐应用。本产品的核心竞争优势在于：

1. **场景整合**：一个 App 覆盖假期家庭娱乐全场景，降低用户切换成本
2. **家庭空间**：以"家庭"为社交单元，而非个人，更贴合家庭使用场景
3. **全龄友好**：从儿童到祖辈的全年龄覆盖，特别是老年用户的极简体验
4. **本地化**：深度中文本地化，融入中国家庭文化和节日元素
5. **安全管控**：内置儿童保护机制，解决家长的核心顾虑

# 交付物四：PRD 产品需求文档（P0-04）

## 4.1 产品概述

### 4.1.1 产品名称
**HomeFun（家庭乐）** — 一站式家庭假期居家娱乐平台

### 4.1.2 产品定位
面向中国家庭的一站式移动娱乐应用，整合多人互动游戏、影音共享和家庭活动规划，让全家人（从儿童到祖辈）都能在同一平台上享受假期时光。

### 4.1.3 目标用户
- **主要用户**：30-45 岁的家庭管理者（家长）
- **次要用户**：8-16 岁青少年、55-70 岁祖辈
- **使用场景**：周末/假期家庭聚会、日常家庭互动、孩子课余时间管理

### 4.1.4 产品愿景
让每一个家庭假期都充满欢笑与温馨，成为"家庭娱乐的第一选择"。

### 4.1.5 成功指标（KPI）

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 家庭创建数 | 上线 3 个月内 10,000+ 家庭 | 后台统计 |
| 日活家庭数（DAF） | 上线 3 个月内 2,000+ | 后台统计 |

# 交付物五：技术选型评估报告（P0-05）

## 5.1 前端框架选型

### 5.1.1 候选方案对比

| 维度 | Flutter | React Native | 原生开发 |
|------|:-------:|:------------:|:-------:|
| **编程语言** | Dart | JavaScript/TypeScript | Swift / Kotlin |
| **性能** | ⭐⭐⭐⭐⭐（自渲染引擎） | ⭐⭐⭐⭐（JSI 新架构） | ⭐⭐⭐⭐⭐ |
| **跨平台一致性** | ⭐⭐⭐⭐⭐（完全一致） | ⭐⭐⭐⭐（大部分一致） | ⭐（各平台独立） |
| **开发效率** | ⭐⭐⭐⭐（热重载，丰富组件） | ⭐⭐⭐⭐（热重载，JS 生态） | ⭐⭐（双倍工作量） |
| **包体积** | ~8-10 MB | ~4 MB | 最小 |
| **社区生态** | ⭐⭐⭐⭐（快速增长） | ⭐⭐⭐⭐⭐（成熟庞大） | ⭐⭐⭐⭐⭐ |
| **学习曲线** | 中等（需学 Dart） | 低（JS 开发者友好） | 高（双语言） |
| **动画/游戏支持** | ⭐⭐⭐⭐⭐（内置动画引擎） | ⭐⭐⭐（依赖第三方） | ⭐⭐⭐⭐⭐ |
| **适合本项目的程度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |

### 5.1.2 选型结论

> **推荐方案：Flutter**
>
> **理由**：
> 1. 自渲染引擎保证跨平台一致性，对游戏动画支持优秀
> 2. 丰富的内置动画组件，特别适合游戏模块（你画我猜等需要流畅画布）
> 3. 单一代码库覆盖手机+平板，开发效率高
> 4. 2026 年 Flutter 生态已非常成熟，社区活跃
> 5. 小团队一人维护双平台可行性高
>
> **风险缓解**：Dart 语言学习成本可通过 1 周集中学习解决；包体积可通过代码分割优化。

### 5.2.1 候选方案对比

| **WebSocket 支持** | ⭐⭐⭐⭐⭐（原生） | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **实时通信** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **TypeScript 生态** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| **团队技能匹配** | 视团队背景 | 视团队背景 | 视团队背景 |

### 5.2.2 选型结论

> **推荐方案：Node.js + NestJS**

> 3. NestJS 提供企业级架构（模块化、依赖注入、守卫等）

> 5. 小团队上手快，社区资源丰富

## 5.3 数据库选型

### 5.3.1 候选方案对比

| 维度 | PostgreSQL | MySQL | MongoDB |
|------|:----------:|:-----:|:-------:|
| **关系型数据支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **JSON 支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **实时特性** | ⭐⭐⭐⭐（LISTEN/NOTIFY） | ⭐⭐⭐ | ⭐⭐⭐⭐（Change Streams） |
| **全文搜索** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **扩展性** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **免费/开源** | ✅ | ✅ | ✅ |

### 5.3.2 选型结论

> **推荐方案：PostgreSQL（主库）+ Redis（缓存/实时）**
>
> **理由**：
> 1. PostgreSQL 的 JSONB 类型灵活存储游戏配置、活动模板等半结构化数据
> 2. 关系型数据模型适合用户、家庭、成员等实体关系
> 3. LISTEN/NOTIFY 支持实时事件推送
> 4. Redis 用于：会话缓存、游戏房间状态、排行榜、消息队列
> 5. 两者组合是业界成熟方案，运维资料丰富

# 交付物七：MVP 范围界定（P0-07）

## 7.1 MVP 定义

**MVP 目标**：以最小成本验证"家庭一站式娱乐"的核心价值假设，上线后获取真实用户反馈。

**核心假设验证**：
1. 家庭用户是否愿意使用"一个 App 管理所有家庭娱乐"的模式？
2. 多人互动游戏是否是家庭场景下的高频需求？
3. 家庭空间（成员管理）是否是有效的用户粘性机制？

## 7.2 MVP 功能清单

### ✅ 包含在 MVP 中

| 模块 | 功能 | 优先级 | 理由 |
|------|------|:-----:|------|
| **用户系统** | 手机号注册/登录 | P0 | 基础门槛 |
| **用户系统** | 微信登录 | P0 | 降低注册门槛 |
| **用户系统** | 个人资料（昵称/头像） | P1 | 基本社交需求 |
| **家庭空间** | 创建/加入家庭 | P0 | 核心差异化 |
| **家庭空间** | 邀请码加入 | P0 | 最简加入方式 |
| **家庭空间** | 成员列表 | P0 | 基础功能 |
| **游戏大厅** | 创建/加入游戏房间 | P0 | 游戏入口 |
| **游戏** | 问答大挑战（内置题库） | P0 | 低门槛高参与 |
| **游戏** | 你画我猜 | P0 | 聚会经典 |
| **影音** | 视频播放器 | P0 | 基础影音能力 |
| **影音** | 内容浏览（分类） | P0 | 内容发现 |
| **活动规划** | 家庭日历（月视图） | P0 | 活动规划核心 |
| **活动规划** | 创建/编辑活动 | P0 | 基础功能 |
| **通用** | 应用首页/导航 | P0 | 体验基础 |
| **通用** | 消息通知（基础） | P1 | 用户触达 |

### ❌ 不包含在 MVP 中（后续迭代）

| 功能 | 移除理由 | 计划版本 |
|------|---------|:-------:|
| 棋类对战 | 开发成本高，非核心验证 | v1.1 |
| 音乐播放器 | 非核心验证点 | v1.1 |
| 同步播放/投屏 | 技术复杂度高 | v1.2 |
| 儿童模式 | 重要但非 MVP 验证点 | v1.1 |
| 任务打卡/积分系统 | 增强粘性，非核心 | v1.2 |
| 家庭投票 | 锦上添花 | v1.3 |
| 搜索与推荐 | 内容量不足时无意义 | v1.2 |
| 自定义题库 | 内置题库足够 MVP | v1.1 |
| 主题设置 | 非核心 | v1.3 |

## 7.3 MVP 成功标准

| 指标 | MVP 目标 | 说明 |
|------|---------|------|
| 家庭创建数 | 500+（上线 2 周） | 验证需求存在 |
| 游戏局数 | 2,000+（上线 2 周） | 验证游戏需求 |
| 次日留存率 | ≥ 30% | 验证产品粘性 |
| 7 日留存率 | ≥ 15% | 验证长期价值 |
| NPS 评分 | ≥ 30 | 验证用户满意度 |
| 应用商店评分 | ≥ 4.0 | 验证基本体验 |
